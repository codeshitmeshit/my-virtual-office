# Agent 协作协议

## 模式

| 模式 | 含义 | 产出 |
|------|------|------|
| 咨询 | 给判断和方案，不动手改 | 建议、方案、风险评估 |
| 执行 | 动手改文件、跑命令、落地结果 | 修改路径、命令输出摘要、证据 |
| 复核 | 检查执行结果，指出风险和遗漏 | 通过/不通过、风险清单、建议 |

## 协作规则

1. **唯一 owner**：每个任务只有一个执行 owner，避免多人改同一块。
2. **模式明确**：主任务单必须标注模式（咨询/执行/复核），会议讨论不默认获得执行授权。
3. **工作区/文件范围**：任务单注明绝对路径，跨系统协作时说清文件在哪个工作区。
4. **完成标准**：任务单包含具体完成条件，不含糊。
5. **交付物**：执行方返回修改路径 + 内容摘要 + 执行证据（命令输出、文件差异等）。
6. **证据摘要**：不要求完整日志，提供关键输出片段即可。
7. **验收人**：每个任务指定一个验收人，负责判断是否完成。
8. **会议 vs 执行**：会议讨论阶段不默认获得执行授权；只有明确标记为执行模式时才能改文件或跑命令。

## 路线

Hermes 和 OpenClaw 是两套平行 agent 系统，各自有独立运行时、工具链、工作区、会话和记忆，通过 Virtual Office 互相通信和协作。不是父子关系。
