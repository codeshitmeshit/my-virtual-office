模块职责汇总

关键观察

项目采用"单文件巨型模块"模式：server.py（10K+ 行）和 game.js（15K+ 行）各自承担了本可拆分为多个模块的职责。这种设计适合小型团队快速迭代，但会增加新人的学习成本。

模块边界清晰度：虽然单个文件很大，但各模块职责边界清晰 — server.py 负责后端，game.js 负责渲染，chat.js 负责聊天，gateway_presence.py 负责状态推导，彼此之间通过 HTTP API 或 WebSocket 通信。
