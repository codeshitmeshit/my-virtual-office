# Weather Glass Furniture Requirement

## 背景

“天气玻璃”是一个沿用窗户设计逻辑的环境氛围型功能家具。它面向无窗、弱窗或缺少好窗景的居住空间，通过类似落地窗的玻璃装置，把实时外部天气以光影、色彩和动态状态带入室内。

该产品不定位为天气 App、智能屏或数据面板，而是一个“像窗户一样存在”的空间家具。用户不需要主动操作，主要通过路过、停留和日常居住时的被动感知，获得外部天气连接感。

## 目标用户

- 居住在无窗、弱窗、内向房、地下室、小公寓、封闭书房等空间的用户。
- 对室内空间氛围、智能家居和长期居住体验有要求的用户。
- 第一版优先验证家庭居住空间，不优先覆盖办公室、酒店、展厅等商业空间。

## 产品目标

- 让无窗或弱窗空间获得“看见外部天气”的感受。
- 减少封闭空间的压迫感和与外部环境断开的感受。
- 保持家具/窗户式的空间融入感，避免像电子屏或天气应用。
- 通过真实天气映射建立稳定、可理解的天气氛围体验。

## 核心范围

- 产品形态接近假窗或电子落地窗，适合挂墙或嵌墙。
- 默认展示实时外部天气氛围。
- 天气表达采用真实映射：下雨表现雨，阴天变暗，雾天表现雾感，晴天表现光照感。
- 信息表达几乎无文字，主要依赖光影、颜色和动态。
- 使用方式以被动感知为主，无需用户主动操作。
- 第一版不承担明确提醒功能，功能性来自空间感知和居住体验改善。

## 非目标

- 不做完整天气 App。
- 不做明显的信息面板、控件界面或多城市天气查询。
- 不以出门提醒、通勤建议、空气质量警报作为第一版核心能力。
- 不替代真实窗户的采光、通风或安全功能。
- 不优先服务商业展示空间。

## 关键约束

- 所有表达必须遵守“像窗户”的设计原则。
- 天气信息沿用窗户逻辑呈现，避免图表、列表、按钮和复杂交互。
- 第一版应优先验证“空间外部连接感”而不是“天气数据完整性”。
- 视觉状态必须足够克制，不能破坏家居空间的长期可接受性。

## 已知结论

- 产品定位：环境氛围型功能家具。
- 产品形态：假窗/电子落地窗。
- 核心价值：改善无窗/弱窗居住空间的外部天气感知。
- 天气策略：实时外部天气真实映射。
- 信息强度：几乎无文字。
- 使用方式：被动感知。
- 第一版边界：不做完整天气 App，不做明确提醒。
