# 项目定时任务 Phase 5 方案评审

## 评审结论

Phase 5 可以进入 checklist 确认阶段。需求边界清楚：它是体验完善和回归加固，不改变 Phase 1-4 的调度、派发、历史和异常语义。

当前没有阻塞性产品问题或技术问题。

## 产品评审

### 已明确

- 推荐对象是长期项目或用户明确标记为长期的项目。
- 推荐只提示创建项目定时任务，不自动创建、不自动启用。
- 用户接受推荐后应复用已有项目定时任务创建表单。
- 本阶段要补齐中英文文案，避免中文硬编码影响英文 UI。
- 本阶段要覆盖完整链路回归，确保 Phase 1-4 不退化。

### 产品边界

- 不做复杂的长期项目智能识别。
- 不新增办公室级自动化平台概念。
- 不引入外部日历、通知偏好或自动重试。
- 不改变项目定时任务的异常提示规则：控制面板仍只提示 failed 和 intervention_required。

### 推荐体验建议

- 第一版优先显式标记：项目创建/编辑表单提供“长期项目”或等价开关。
- 当项目被标记为长期项目且没有项目定时任务时，在项目详情页显示推荐入口。
- 如果项目已有定时任务，推荐不再重复展示，避免噪音。
- 推荐入口直接打开现有“新建项目定时任务”表单，并保留用户确认保存的动作。

## 技术评审

### 数据模型

建议在项目模型中增加一个轻量字段，例如 `longRunning` 或 `longTermProject`，用于显式标记长期项目。旧项目没有该字段时按 `false` 处理。

该字段应随 markdown 项目存储持久化，并在项目 summary/detail API 返回，供列表和详情页判断是否显示推荐。

### UI 接入

优先接入现有项目创建/编辑表单和项目详情页：

- 创建/编辑项目时可设置长期项目标记。
- 项目详情页在“项目定时任务”区域附近展示推荐入口。
- 推荐入口复用 `createProjectCronPrompt()` 或同等已有表单，不新增第二套创建逻辑。

### 本地化

需要将项目定时任务相关硬编码文案迁移到 locale，至少覆盖：

- 项目定时任务面板标题、说明、空状态、暂停/恢复。
- 新建/编辑/保存/删除/立即运行相关提示。
- 执行历史标题、状态标签和空状态。
- 长期项目推荐提示、按钮和说明。
- 重复触发说明。
- 全局定时任务页项目 cron 创建入口和提示。

### 兼容性

- 旧项目没有长期项目字段时应正常加载。
- 旧项目没有 `scheduledCronHistory` 时继续显示空状态。
- 旧项目没有项目 cron 绑定时不显示推荐之外的异常状态。
- 普通 Agent cron 不受长期项目推荐影响。

### 测试可行性

可以通过服务端存储测试覆盖长期项目字段持久化和旧项目兼容，通过前端静态检查覆盖 JS 语法，通过项目定时任务既有测试覆盖 Phase 1-4 回归，并通过 8090 live 验收覆盖推荐入口和“不自动创建 cron”。

## 风险与缓解

- 风险：推荐提示过多造成干扰。
  - 缓解：只对长期项目且当前没有项目定时任务时展示。
- 风险：文案迁移遗漏导致英文 UI 仍显示中文。
  - 缓解：用 `rg` 扫描新增/相关硬编码文案，并人工检查中英文 locale。
- 风险：长期项目字段迁移影响旧项目读取。
  - 缓解：字段默认 false，存储读写保持可选。
- 风险：推荐误触发创建 cron。
  - 缓解：推荐按钮只打开表单，只有用户保存表单才调用创建 API。

## 结论

Phase 5 范围适合独立子需求执行。Checklist 确认前不生成 todolist。
