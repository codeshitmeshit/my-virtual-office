# 项目定时任务 Phase 5：长期项目推荐、体验打磨和回归加固

## 背景

项目定时任务父需求已完成 Phase 1 至 Phase 4：

- Phase 1 已支持复用现有 Cron 并保存项目绑定。
- Phase 2+3 已支持项目详情页配置、全局定时任务总览和真实项目派发。
- Phase 4 已支持项目定时执行历史、控制面板异常提示和调度诊断。

剩余 Phase 5 聚焦长期项目使用体验：让用户在合适场景更容易发现项目定时任务，同时补齐中英文文案、兼容性和完整回归，避免核心功能已实现但入口不明显、文案不一致或长期使用中回归不足。

## 目标用户

- 项目创建者或项目负责人。
- 维护长期项目、周期性任务、日报/同步/巡检类项目的用户。

## 目标

- 在长期项目设置流程中推荐创建项目定时任务。
- 推荐只作为提示和入口，不自动创建、不自动启用任何 cron。
- 补齐项目定时任务相关 UI 文案的中英文本地化。
- 加固旧项目兼容和 Phase 1-4 关键链路回归。
- 保持“项目定时任务”产品心智，不扩展成通用自动化规则平台。

## 范围

### 包含

- 为项目增加或复用一个轻量的长期项目标记/判断入口。
- 在项目创建、编辑或项目详情页中，对长期项目展示创建项目定时任务的推荐入口。
- 推荐入口跳转或打开已有项目定时任务创建表单。
- 明确提示“不会自动启用，需用户确认创建”。
- 补齐新增和既有项目定时任务相关 UI 文案的中英文 locale。
- 旧项目缺少长期项目字段、定时任务字段、历史字段时正常加载。
- 补充 Phase 1-4 的综合回归测试和 8090 live 验收清单。

### 不包含

- 自动判断所有项目是否长期项目的复杂算法。
- 自动创建或自动启用定时任务。
- 通用办公室自动化规则平台。
- 外部日历、通知偏好、自动重试策略。
- 重做 Phase 1-4 已完成的核心能力。

## 已知现状

- `app/projects.js` 中项目定时任务 UI、历史 UI 和项目执行 UI 存在较多硬编码中文。
- `app/cron.html` 中项目定时任务相关文案也存在硬编码中文。
- `app/locales/en.json` 和 `app/locales/zh.json` 尚未完整覆盖项目定时任务文案。
- 项目模型已有 `projectExecutionEnabled`、`scheduledCronPaused`、`scheduledCronHistory` 等字段，但未见明确的长期项目推荐字段。
- Phase 4 已将项目执行历史和控制面板异常提示归档为完成。

## 成功标准

- 用户创建或编辑长期项目时能看到项目定时任务推荐。
- 推荐不会在用户未确认时创建 cron，也不会自动启用 cron。
- 用户接受推荐后复用现有项目定时任务创建表单。
- 中英文 UI 下项目定时任务入口、提示、状态、历史和错误文案无明显硬编码遗漏。
- 旧项目正常加载，未配置长期项目和定时任务时没有报错。
- Phase 1-4 的关键能力仍通过自动化测试和 8090 live 验收。
