# Meeting Branch Participant Selection

## 背景

Meeting for AI 已支持用户新建会议，以及 AI 发起会议申请后由用户确认会议配置。当前两个入口的参会人选择都是平铺 AI 列表，用户需要逐个勾选参会 AI。当 AI 数量增加，并且办公室已经按 branch/部门组织 agent 时，逐个选择会变慢，也更容易漏选或误选。

本需求希望在会议参会人选择时增加 branch 维度的快捷勾选能力：用户可以先按 branch 批量选择一组 AI，再手动取消不需要参加的 AI。

## 目标用户

- 主动创建会议的用户。
- 审核 AI 会议申请并调整参会人的用户。
- 需要按团队、部门、业务线快速组织会议的项目负责人。

## 产品目标

1. 在“新建会议”和“AI 会议申请确认”两个入口都支持按 branch 快捷选择参会人。
2. branch 选择是批量选择工具，不改变会议的最终参会人模型；最终参会人仍以用户确认后的 AI 列表为准。
3. 勾选 branch 时选中该 branch 下所有 AI；取消 branch 时取消该 branch 下所有 AI。
4. 用户通过 branch 选择后，可以继续手动取消或重新勾选单个 AI。
5. 当 branch 下仅部分 AI 被选中时，branch 显示半选状态，确保“选了谁”一眼可见。
6. branch 选择不主动改变 Moderator；但如果当前 Moderator 被取消，则自动切换到剩余参会人中的第一个。

## 范围

### 本期包含

- 新建会议表单中的参会人 branch 快选。
- AI 会议申请确认表单中的参会人 branch 快选。
- branch 全选、取消全选、半选状态。
- branch 批量选择后仍可手动调整单个 AI。
- 单个 AI 调整后同步更新 branch 状态。
- Moderator 选项随最终参会人列表保持一致；当前 Moderator 被取消时自动兜底。
- 中英文 UI 文案。

### 本期不包含

- 不改变会议执行顺序、轮次或发言策略。
- 不把会议拆成按 branch 分组运行。
- 不新增后端会议协议字段；后端仍接收最终 participants 数组。
- 不新增 branch 权限、branch 专属会议类型或跨 branch 权限控制。
- 不改变 AI 会议申请自动确认规则。

## 关键约束

- branch 快选必须是辅助选择，不应阻止用户跨 branch 组合参会人。
- 至少两名参会人的既有校验仍保持。
- Moderator 必须是参会人之一。
- 现有没有 branch 或 branch 为 `UNASSIGNED` 的 agent 仍应可被选择。
- 已有会议、会议申请、会议执行、会议结果和行动项闭环不能回归。

## 已澄清产品结论

- 适用入口选择：C，两个入口都支持。
- branch 交互语义选择：C，branch 作为复选框维度，支持全选/取消全选并允许单个 AI 微调。
- branch 部分选中表达选择：A，使用半选状态。
- Moderator 处理选择：B，当前 Moderator 被取消时自动切换到剩余参会人中的第一个。
- 成功标准选择：C，优先保证选中结果清晰可见，其次提升效率。

## 成功标准

- 用户能在新建会议时通过 branch 一键选择某个部门/团队的 AI，并手动移除不需要的 AI。
- 用户能在 AI 会议申请确认时通过 branch 快速调整建议参会人，并手动微调。
- branch 半选状态准确表达部分选中。
- Moderator 始终有效，不会因为取消参会人而出现空选或非法选项。
- 会议创建和申请确认仍以最终 participants 数组为准，后端行为保持兼容。
