# 通用项目执行 通用项目执行开发任务清单

## 执行规则

- 严格按 执行底座、独立审查与最终验收 顺序实施。
- 执行底座 完成自动化测试和浏览器验收并取得用户确认后，才能开始 独立审查与最终验收。
- 每项实现必须使用显式 `executionState` 和 `attemptId`，不得继续依赖列标题推断规范状态。
- 不得把旧版执行者自审、自动模式或拖拽进 Done 作为临时过渡方案。
- 不处理同项目并发任务、worktree 隔离、通用定时自动化、自动 Git 提交或推送。

### TODO-001 扩展项目与任务持久化模型

- 输入：现有 Markdown/JSON 项目结构、通用项目执行 requirement/review。
