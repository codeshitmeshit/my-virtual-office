# App Font Size Scaling

## 背景

当前主应用 `app/` 的基础字号未显式设置，浏览器默认基础字号约为 `16px`。实际 UI 中大量组件使用固定 `px` 字号，例如设置面板、工具栏、项目管理、会议、聊天、Agent 面板和动态弹窗。用户希望在设置处增加一个字体大小设置，让主应用 DOM UI 字体按一定比例变大。

## 目标用户

- 主应用使用者，尤其是觉得当前像素风小字号难以阅读的用户。
- 使用中文界面、低分辨率屏幕、远程桌面或浏览器缩放不方便的用户。

## 目标

- 在主应用设置面板中提供字体大小缩放设置。
- 用户调整后，主应用 DOM UI 字体应按比例变大。
- 设置应作为当前浏览器/当前用户的个人显示偏好持久化，刷新后保留。
- 默认值保持现有视觉，不改变未配置用户的体验。

## 范围

- 主应用页面 `app/index.html`。
- 主应用样式 `app/style.css` 与相关独立样式如 `app/projects.css`。
- 设置加载和保存逻辑 `app/game.js`。
- 中英文文案 `app/locales/en.json`、`app/locales/zh.json`。
- 常见 DOM UI，包括设置面板、工具栏、侧栏、主要弹窗、聊天、项目面板、会议面板等。
- 字体大小档位固定为 `100%`、`110%`、`120%`、`130%`、`150%`。

## 非目标

- 不调整营销页 `website/`。
- 不改变浏览器原生缩放。
- 不重做整体响应式布局或所有固定尺寸。
- 不覆盖办公室画布里的 Agent 名称、状态、家具标签等 canvas 绘制文字。
- 不写入全局办公室配置，不影响其他用户或其他浏览器。
- 不保证用户自定义文本标签的业务字号被强制覆盖；这类字号可能属于内容属性。
- 不在本需求中重构所有内联样式。

## 关键约束

- 当前工作区已有未提交改动，涉及 `app/game.js`、`app/index.html`、`app/style.css`、locale 文件等，后续实现必须避免覆盖无关修改。
- 主应用很多字号是固定 `px`，单纯设置 `body font-size` 无法覆盖。
- `game.js` canvas 文本通过 `ctx.font = '9px Arial'` 等方式绘制，CSS 变量不会自动生效；本期已明确不覆盖这类文字。
- UI 布局中存在大量固定宽高，字体放大后可能导致按钮、侧栏、弹窗溢出。
- 最终验收必须使用 MCP 浏览器工具打开真实页面，检查设置入口、DOM 状态、关键面板和布局，而不能只做静态代码检查。

## 已知结论

- 第一版定义为“DOM UI 字体缩放”而不是“所有文字绝对放大”。
- 产品澄清已确认：范围为 DOM UI；档位为 `100%`、`110%`、`120%`、`130%`，后续按用户要求追加 `150%`；设置为个人偏好；验收范围为设置、工具栏、主要弹窗、聊天、项目、会议等 DOM 面板。
- 设置应保存在 `localStorage` 的展示偏好中，和现有 `vo-display-prefs` 合并较自然。
