# 需求与技术方案评审

## 评审结论

**结论：通过，可进入测试 checklist 确认。**

需求目标明确，字体与产品像素风格匹配，许可证允许商业软件嵌入和再分发。没有阻塞性产品或技术问题。

## 产品评审

### 合理性

- 当前问题不是单纯字号差异，而是 `Press Start 2P` 缺少中文字形后发生系统字体回退。
- 在中文模式中切换到同时覆盖中文和拉丁字符的像素字体，能够从根源上统一混排视觉。
- 仅中文模式切换字体，可避免影响已经稳定的英文界面。

### 范围判断

- 应覆盖所有具备中文界面的页面，不能只修改主页面，否则独立设置页仍会出现明显差异。
- 官网中文内容也应使用同一中文字体策略，但正文可读性和资源体积需要单独验证。
- 代码、路径和技术标识不应为了“统一”而失去等宽特性。

## 技术方案

### 资源接入

1. 从 Fusion Pixel Font 官方 Release 获取 `zh_hans`、比例模式、12px 版本。
2. 优先使用官方提供的 WOFF2；若发布包没有合适格式，再评估 TTF 直接嵌入。
3. 字体文件存放于项目静态资源目录，不使用第三方热链。
4. 增加 `@font-face`，设置稳定的内部字体家族名和 `font-display: swap`。
5. 将 `LICENSE-OFL` 和字体来源说明随字体资源一同归档。

### 字体应用

建议通过 CSS 变量统一控制：

```css
:root {
  --font-pixel-ui: "Press Start 2P", cursive;
  --font-zh-ui: "Fusion Pixel 12px Proportional ZhHans", sans-serif;
}

html[lang="zh"] {
  --font-pixel-ui: var(--font-zh-ui);
}
```

- 现有像素 UI 统一引用变量，避免在大量选择器中重复覆盖。
- `i18n.js` 已在语言切换时更新 `document.documentElement.lang`，因此 CSS 可即时响应。
- 技术内容保留专用 `monospace` 样式，不继承中文 UI 字体变量。

### 页面覆盖

- `app/style.css`：主办公室及动态 DOM 的默认像素字体。
- `app/setup.html`：安装向导内联样式。
- `app/models.html`：模型页内联样式。
- `app/cron.html`：定时任务页标题与表单。
- `website/styles.css`：官网中文模式。
- 动态内联样式：审查 `game.js` 中硬编码的 `Press Start 2P`，必要时改为 CSS 类或变量。

## 风险与处理

### 字体包体积

泛中日韩字体可能显著增加首次加载体积。

处理：

- 只引入简体中文 `zh_hans` 版本。
- 优先 WOFF2。
- 检查 Release 中是否已有按语言和尺寸拆分的产物。
- 设置长期缓存和内容哈希或版本化 URL。
- 不在未确认许可证与字体名称限制前自行子集化。

### 视觉尺寸变化

Fusion Pixel Font 的字面框、基线和字宽不同于当前系统回退字体，可能引起按钮、标签和弹窗尺寸变化。

处理：

- 先保持现有字号，按 MCP 实测结果仅调整发生截断的局部行高或容器尺寸。
- 不进行全局字号重构。
- 在 100% 缩放及窄屏下检查关键页面。

### 字体闪烁

使用 `font-display: swap` 时可能短暂显示回退字体。

处理：

- 字体本地缓存。
- 在中文首屏页面预加载核心 WOFF2。
- 检查字体失败时界面仍可读。

### 许可证合规

OFL 允许免费商用和随软件分发，但要求字体及衍生字体继续使用 OFL。

处理：

- 保留完整 `LICENSE-OFL`。
- 增加来源、版本、下载地址和未修改/修改状态说明。
- 若转换、裁剪或改名，记录构建方式和衍生字体许可证。
- 不单独销售字体。

## 兼容性

- 现代 Chromium、Firefox、Safari 应支持 WOFF2 和 `@font-face`。
- 无法加载 WOFF2 时回退到系统中文无衬线字体。
- 英文模式不请求中文字体是优化目标；至少不得应用中文字体。
- 离线或外网不可用时，自托管字体仍应正常加载。

## 性能与可观测性

- 通过浏览器 Network 面板验证字体请求次数、状态码、传输体积和缓存命中。
- 检查控制台无字体解码、CORS、MIME 或许可证资源路径错误。
- 记录字体文件版本，便于后续升级和回滚。

## 测试策略

- 自动检查静态资源存在、CSS 引用有效和许可证文件存在。
- 所有可视化验收必须使用 MCP 浏览器工具验证。
- 分别验证中文首次加载、英文首次加载、中英运行时切换和字体加载失败。
- 对主页面、独立页面、动态弹窗、表单和窄屏布局做截图或 DOM 计算字体核验。
