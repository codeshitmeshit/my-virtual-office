# 中文界面使用缝合像素字体

## 背景

当前英文界面主要使用 `Press Start 2P` 像素字体。该字体不包含中文字形，切换到中文后浏览器会为中文回退到系统字体，导致中英文在字形风格、字重、基线和字符宽度上差异明显。

## 目标用户

- 使用简体中文界面的 Virtual Office 用户。
- 需要中英文混排且希望保持像素视觉风格的用户。

## 目标

1. 中文模式统一使用 [Fusion Pixel Font（缝合像素字体）](https://github.com/TakWolf/fusion-pixel-font)。
2. 中文和英文字符在中文界面中使用同一字体家族，减少混排差异。
3. 英文模式继续保持现有字体和视觉效果。
4. 字体资源由项目自托管，不依赖运行时访问第三方字体服务。
5. 满足 SIL Open Font License 1.1 的再分发要求。

## 功能范围

- 主办公室页面。
- 安装向导。
- 模型与提供商设置页。
- 定时任务管理页。
- 官网中可切换为中文的内容。
- 动态生成的按钮、菜单、弹窗、侧栏状态和表单控件。
- 页面加载、首次语言检测和运行时语言切换。

## 字体选择

- 字体项目：TakWolf/fusion-pixel-font。
- 字形版本：`zh_hans`。
- 宽度模式：比例模式。
- 基础像素尺寸：12px 版本。
- 使用官方发布包中已有的 Web 字体格式；如需自行转换或裁剪，产物继续遵守 OFL-1.1，并检查保留字体名称限制。

## 需求点

- `REQ-001`：当 `html.lang` 为 `zh` 时，普通界面文字使用 Fusion Pixel Font。
- `REQ-002`：中文模式中的拉丁字母、数字和中文应来自同一字体家族，除明确保留的代码与等宽内容。
- `REQ-003`：当 `html.lang` 为 `en` 时，现有英文像素字体及版式不发生变化。
- `REQ-004`：代码、终端、路径、令牌、模型 ID 等技术内容继续使用现有等宽字体。
- `REQ-005`：字体文件随项目部署，自托管加载；字体加载失败时提供可读的系统中文字体回退。
- `REQ-006`：主界面及独立页面采用一致的中文字体策略。
- `REQ-007`：语言切换后无需刷新即可更新字体。
- `REQ-008`：字体接入不得造成按钮截断、文本重叠、弹窗越界或主要布局抖动。
- `REQ-009`：仓库中保留 Fusion Pixel Font 的 OFL-1.1 许可证及来源说明。
- `REQ-010`：字体资源应使用缓存策略，避免每次页面访问重复下载。

## 非目标

- 不重新设计字号体系、颜色、间距或整体 UI。
- 不修改 Canvas 内由程序绘制的非 DOM 像素文字，除非现有 Canvas 已直接依赖网页字体且无需额外重构。
- 不替换英文模式的 `Press Start 2P`。
- 不对第三方字体进行字形设计或人工修字。
- 不在本期处理繁体中文、日文或韩文字形版本。

## 约束

- 字体使用 SIL Open Font License 1.1。
- 不从 GitHub、CDN 或字体演示站点热链生产字体。
- 当前工作区存在其他未提交汉化修改，实施时必须保留并兼容这些修改。
- 用户明确确认 checklist 前不得进入实施。

## 已知结论

- Fusion Pixel Font 允许免费商用、嵌入、修改和随软件分发。
- 字体本身及衍生字体需继续遵守 OFL-1.1，且不能作为独立商品出售。
- 项目说明建议普通 UI 优先使用比例模式，其基线和混排观感优于等宽模式。
