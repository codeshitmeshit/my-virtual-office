# Functional Furniture Bookshelf Archive

## 背景

当前家具编辑器中已有普通家具类目和书架家具。用户希望新增一个“功能性家具”类目，用于承载可以被点击并进入业务功能的家具。本期先引入一个功能性家具：书架。

## 目标用户

- 使用家具编辑器布置虚拟办公室的用户。
- 在办公室场景中通过家具快速进入档案内容的用户。

## 产品目标

- 在家具编辑器中新增“功能性家具”类目。
- 本期只在该类目中提供书架。
- 书架作为可交互入口，支持绑定一个具体档案。
- 用户点击书架后可以查看已绑定档案，或绑定/更换档案。

## 已澄清结论

- 功能性家具定位为“可交互入口”类目。
- 本期只引入一个功能性家具：书架。
- 书架主要日常行为是查看档案，绑定档案是配置行为。
- 一个书架绑定一个具体档案。
- 点击“查看档案”后，直接打开已绑定档案的详情页。
- 已绑定状态下，“绑定档案”文案应变为“更换档案”。
- 同一个档案允许被多个书架绑定，作为多个快捷入口。
- 书架绑定信息只在编辑模式或悬停/选中时显示。
- 未绑定时，点击书架仍显示两个操作，但“查看档案”置灰不可点，并显示未绑定状态。
- 本期不展示或预留其他功能性家具。

## 范围

### 本期包含

- 家具编辑器新增“功能性家具”类目。
- 功能性家具类目中出现书架。
- 书架可被放置、移动和保存，与现有可编辑家具一致。
- 点击书架时出现两个操作：
  - 查看档案。
  - 绑定档案；已绑定时显示为更换档案。
- 未绑定状态下，“查看档案”不可用，并有明确未绑定提示。
- 绑定成功后，书架点击“查看档案”进入对应档案详情页。
- 编辑模式或悬停/选中状态可识别当前绑定档案。

### 本期不包含

- 除书架外的其他功能性家具。
- 功能性家具类目的未来家具占位或敬请期待项。
- 档案库、档案分类或多个档案绑定。
- 限制一个档案只能绑定一个书架。
- 新建档案能力，除非现有绑定流程已经支持。

## 关键状态

- 未绑定：书架存在但没有档案关联；查看档案置灰，绑定档案可用。
- 已绑定：书架关联一个具体档案；查看档案可用，绑定入口显示为更换档案。
- 绑定目标不可用：档案被删除、不可访问或失效时，应有可理解反馈，并允许重新绑定。

## 成功标准

- 用户能在家具编辑器中明确找到“功能性家具”类目。
- 用户能从该类目添加书架。
- 用户能完成书架与具体档案的绑定。
- 用户能通过点击已绑定书架直接打开档案详情。
- 未绑定书架不会给用户造成“入口坏了”的误解。
- 现有普通书架或其他家具编辑能力不发生回归。
