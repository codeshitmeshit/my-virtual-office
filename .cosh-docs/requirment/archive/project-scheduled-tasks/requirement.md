# 项目定时任务需求

## 背景

Virtual Office 目前已经有项目工作流和面向 Agent 的 Cron Manager。对于长期项目，项目负责人不应该每天都手动点击继续执行；系统需要一种项目级的定时能力，定期唤醒项目、继续项目工作，或者重复执行某个项目内的周期性任务。

当前 Cron Manager 可以定时给 Agent 发送 prompt，但它没有绑定 `projectId`，不会直接启动项目 workflow，也不会把执行记录沉淀到项目详情里。因此它只能作为临时替代方案，不能满足真正的项目级定时任务需求。

## 产品概念

功能名称使用 **项目定时任务**。

这个概念不要做成泛化的“自动化规则平台”，也不要只做成一个“每日自动推进”开关。它的边界是：

- 归属于某个项目。
- 可以启动整个项目 workflow。
- 可以重复执行某个指定项目任务。
- 项目详情页展示执行记录。
- 控制面板只提示失败或需要人工介入的异常状态。

## 目标用户

- 主要用户：项目创建者或项目负责人。
- 可配置项目：必须有明确负责人或绑定 Agent，避免失败后无人接收异常和处理提示。

## 目标

- 让长期项目可以按计划自动继续推进。
- 让项目负责人可以配置周期动作，例如每日同步、日报生成、阻塞检查。
- 让定时执行能带来真实项目推进，例如 workflow 启动、任务开始、任务完成、进入 review。
- 让异常状态可见，但不让正常执行记录污染控制面板。
- 将实现拆成 5 个 phase，降低一次编码量过大导致出错的风险。

## 范围

### 本期包含

- 复用现有 Cron 能力作为底层调度器，在 cron job 上增加项目绑定语义，必须能绑定 `projectId`。
- 调度类型：
  - cron 风格固定时间。
  - 每隔固定时间循环执行。
  - 指定时间一次性执行。
- 定时目标：
  - 整个项目 workflow。
  - 指定项目任务。
- 触发整个项目 workflow 时的默认行为：
  - 如果项目没有运行中的 workflow，则自动启动。
  - 如果项目已有运行中的 workflow，则跳过本次执行并记录原因。
- 指定任务的重复执行：
  - 选中的任务可以作为固定周期任务重复执行，例如同步、报告、检查任务。
- 项目详情页执行历史：
  - 记录触发、启动、跳过、失败、完成、需要人工介入等状态。
- 全局定时任务页总览：
  - `/cron.html` 需要能看到项目绑定 cron，并标明项目、目标类型和目标任务。
  - 全局定时任务页用于系统级可见性和排查，不应只显示普通 Agent cron。
- 控制面板提示：
  - 只提示失败和需要人工介入。
- 项目级暂停/恢复：
  - 项目负责人可以暂停或恢复该项目下的全部定时任务。
- 长期项目推荐：
  - 新建或标记长期项目时，可以推荐创建定时任务，但不能未经用户确认自动启用。
- 新增 UI 文案需要支持中英文。

### 本期不包含

- 完整的办公室级通用自动化规则平台。
- 跨项目自动化链路。
- 外部日历集成。
- 每个定时任务的复杂通知偏好设置。
- 自动重试策略；第一版只要求记录失败并提示人工处理。
- 超出项目定时任务需要的 provider-neutral scheduler 架构重构。

## 已澄清的产品结论

- 核心问题：长期项目需要自动唤醒继续推进，同时部分项目有固定周期动作。
- 功能心智：项目定时任务。
- 触发动作：默认启动整个项目 workflow，同时支持指定项目任务。
- 主要使用者：项目负责人或项目创建者。
- 第一版范围：支持时间规则、项目/任务目标、Agent/消息等必要配置。
- 成功标准：项目状态真实推进，而不只是发出提醒。
- workflow 已运行时：跳过并记录，不重复启动。
- 执行可见性：项目详情页展示完整执行记录。
- 失败处理：项目页标记异常，并在控制面板提示。
- 使用边界：项目必须有负责人或绑定 Agent。
- 默认策略：长期项目推荐创建，但不自动启用。
- 控制面板噪音控制：只显示失败和人工介入。
- 项目级控制：支持暂停/恢复项目全部定时任务。
- 全局可见性：项目绑定 cron 也必须出现在全局定时任务页，用户能知道系统当前有哪些自动任务。

## Phase 拆分

### Phase 1：扩展现有 Cron 支持项目绑定

先复用现有 Cron job 的调度、启停、last/next run 等基础能力，扩展其元数据，使 cron job 可以绑定项目、指定项目目标并完成必要校验。该阶段只保存和管理项目绑定信息，不做真实项目执行。

### Phase 2：项目详情页配置体验与全局定时任务总览

在项目详情页提供定时任务创建、编辑、启停、删除、立即运行和项目级暂停/恢复；同时在全局定时任务页 `/cron.html` 展示项目绑定 cron，让用户能从系统层面看到所有定时任务。

### Phase 3：调度派发与项目执行绑定

处理到期定时任务，将其绑定到现有项目 workflow 或指定任务执行流程。

### Phase 4：执行历史与控制面板异常提示

记录项目级执行历史，并且只把失败和需要人工介入的状态提示到控制面板。

### Phase 5：长期项目推荐、体验打磨和回归加固

增加长期项目推荐创建定时任务，完善本地化、兼容性和回归测试。

## 成功标准

- 项目负责人可以创建、编辑、暂停、恢复、立即运行和删除项目定时任务。
- 全局定时任务页可以看到项目绑定 cron，并清楚区分普通 Agent cron 与项目 cron。
- 没有负责人且没有绑定 Agent 的项目不能配置定时任务。
- 到期的项目级 workflow 定时任务，在项目空闲时能启动 workflow。
- 到期的项目级 workflow 定时任务，在 workflow 已运行时能跳过并记录原因。
- 到期的指定任务定时任务能启动对应项目任务。
- 项目详情页展示定时执行记录。
- 控制面板只提示失败和人工介入。
- 项目级暂停能阻止该项目下所有定时任务执行。
- 长期项目只推荐创建定时任务，不自动启用。
