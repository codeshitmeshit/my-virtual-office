# Meeting for AI Phase 5：Agent 忙碌冲突、任务暂停与并发

## 背景

Meeting for AI 已完成 Phase 1-3 的可执行会议基础、用户实时控制与争议裁决，并完成 Phase 4 的 AI 发起会议申请、用户确认和上下文快照能力。当前剩余关键风险是：会议开始时可能打断正在执行项目任务或其他工作的 Agent，也可能让同一个 Agent 被重复加入多场会议。

Phase 5 目标是让会议安全接入真实 Agent 工作状态，在用户可理解、可选择、可追踪的前提下处理忙碌冲突、任务暂停、恢复和会议并发。

## 目标用户

- 在 Virtual Office 中管理多个 AI Agent 的用户。
- 需要临时召集忙碌 Agent 参与会议的用户。
- 正在执行项目任务、普通工作或会议中的 Agent。
- 需要判断是否等待、更换或强制加入的会议发起者。

## 产品目标

1. 会议创建或确认时能够识别参会 Agent 的忙碌状态和会议占用冲突。
2. 用户可以在冲突场景中选择等待、更换参会者、强制加入或轻量预约。
3. 系统在中高风险冲突下提供 advisory turn 推荐，帮助用户理解打断影响。
4. Agent 参会前的任务和状态被记录；参会期间任务不会被静默破坏。
5. 会议正常结束、取消或失败后，原任务恢复具备幂等和人工补救路径。
6. 多场会议可并行，但同一个 Agent 同一时间只能参加一场会议。

## 范围

### 忙碌冲突检测

- 检测参会 Agent 是否正在执行项目任务、普通持续工作、provider 调用、外部等待或另一场会议。
- 区分空闲、低风险忙碌、中高风险忙碌和已被会议占用。
- 冲突检测结果需要展示给用户，不能静默打断或重复占用 Agent。

### 冲突处理选择

用户在冲突场景中可以选择：

- **等待**：保留会议准备状态，等待 Agent 空闲后再继续。
- **更换参会者**：用其他 Agent 替代忙碌 Agent。
- **强制加入**：用户明确确认打断风险后，将忙碌 Agent 拉入会议。
- **轻量预约**：把会议标记为稍后提醒或到空闲后再尝试，不做完整日程系统。

### Advisory Turn

- Advisory turn 定位为推荐决策：它可以推荐等待、更换或强制加入，但不自动执行。
- 中高风险冲突默认触发 advisory turn。
- 低风险冲突可以不默认触发，但用户准备强制加入且尚无建议时，应先尝试获取建议。
- Advisory 内容应优先说明：
  - Agent 当前忙碌原因。
  - 预计可用时间或不确定性。
  - 打断风险。
  - 恢复说明或 resume notes。
  - 推荐的冲突处理动作。
- Advisory 失败、超时或建议不清晰时，用户仍可继续，但界面必须说明无法获取建议；强制加入需要二次确认。

### 轻量预约

- 预约作为冲突处理选项，不扩展为完整日历或强排程系统。
- 主要用途是避免打断忙碌 Agent。
- 预约对 Agent 是部分占用或可见安排：显示未来有预约，但不阻止当前任务继续。
- 到了预约时间或触发条件时，如果 Agent 仍忙碌，应重新进入冲突处理，而不是自动强制加入或自动取消。
- 文案必须表达为“稍后提醒/到时尝试”，不能暗示到时一定能开会。

### 任务暂停与恢复

- Agent 正式参会前，记录原任务、原状态、冲突处理原因和恢复信息。
- 支持 provider 能力范围内的真实暂停。
- 无法真实暂停时使用逻辑暂停，阻止系统继续派发新工作，并明确展示限制。
- 会议正常结束、取消或失败后尝试恢复可恢复任务。
- 恢复操作必须幂等；恢复失败时显示明确告警和人工补救状态。

### 并发会议

- 系统允许多场会议并行。
- 同一 AI 同一时间只能参加一场会议。
- 第二场会议包含已参会 AI 时，必须进入冲突状态。
- 不包含冲突 AI 的其他会议仍可正常并行。

### UI 与可观测性

- 会议中心、会议详情和办公室 Agent 状态需要展示冲突、等待、预约、暂停、恢复和占用状态。
- 强制加入、恢复失败和 advisory 失败都必须有可理解文案。
- 用户选择和关键状态变化需要可追踪。

## 非目标

- 不做完整日历、多人排期或长期会议日程系统。
- 不允许 advisory turn 自动暂停任务、释放占用或强制加入会议。
- 不允许 Agent 自报状态替代服务端冲突检测。
- 不支持同一个 AI 同时参加多场会议。
- 不自动裁决是否打断忙碌 Agent；最终选择仍由用户确认或显式策略决定。

## 已确认产品决策

- Advisory turn 的产品定位是推荐决策，不自动执行。
- 中高风险冲突默认触发 advisory turn。
- 冲突面板优先展示忙碌原因、预计可用时间、打断风险和恢复说明。
- Advisory 失败时允许用户继续，但强制加入需要二次确认。
- Phase 5 成功标准优先保护安全性。
- Phase 5 支持轻量预约：冲突时用户可以选择等 Agent 空闲后提醒或再开，不做完整日程系统。
- 预约主要用于避免打断忙碌 Agent。
- 到预约时间或触发条件时，如果 Agent 仍忙碌，应再次进入冲突处理。
- 预约对 Agent 是部分占用或可见安排，不阻止当前任务继续。

## 成功标准

- 会议不会静默破坏正在执行的任务。
- 同一个 Agent 不会同时参与多场会议。
- 用户能理解冲突原因、打断影响和可选处理方式。
- 强制加入前有明确风险说明和用户确认。
- 任务暂停和恢复路径可追踪、幂等，并能在失败时提供人工补救状态。
- 普通聊天、项目任务创建、项目执行和已完成 Phase 1-4 能力不回归。
