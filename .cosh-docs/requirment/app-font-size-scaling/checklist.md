# App Font Size Scaling Checklist

确认状态：已确认

## 产品边界

- 本期只验收 DOM UI 字体缩放。
- 本期不验收 canvas 绘制文字缩放，例如办公室画布里的 Agent 名称、状态、家具标签。
- 字体大小档位固定为 `100%`、`110%`、`120%`、`130%`、`150%`。
- 设置只影响当前浏览器/当前用户，不写入全局办公室配置。
- 最终验收必须使用 MCP 浏览器工具检查真实页面。

## Checklist

### CHK-001: 设置入口和文案可见

- 关联需求点：在设置处增加个人 UI 字体大小设置。
- 验证方法：用 MCP 浏览器工具打开主应用页面，点击 `☰ Menu`，查看设置面板。
- 预期结果：设置面板中出现 `UI font size` / `界面字体大小` 控件；文案表达为界面/UI 字体大小，不暗示 canvas 或所有可见文字都会缩放。

### CHK-002: 固定档位完整且默认正确

- 关联需求点：字体大小使用固定档位 `100%`、`110%`、`120%`、`130%`、`150%`。
- 验证方法：用 MCP 浏览器工具检查字体大小控件的可选项。
- 预期结果：控件只提供 `100%`、`110%`、`120%`、`130%`、`150%` 五个主要档位；首次打开或无本地偏好时默认选中 `100%`。

### CHK-003: 字体缩放即时应用到设置面板

- 关联需求点：用户调整后 DOM UI 字体按比例变大。
- 验证方法：用 MCP 浏览器工具在设置面板内把字号从 `100%` 调到 `120%`，观察设置标题、标签、输入框、按钮文本的计算字号或视觉变化。
- 预期结果：设置面板内主要文本无需刷新即可变大；页面没有白屏或脚本错误。

### CHK-004: 工具栏随缩放变化

- 关联需求点：主应用常用 DOM UI 受字体缩放影响。
- 验证方法：用 MCP 浏览器工具在 `100%` 和 `120%` 间切换，检查底部/顶部工具栏按钮，例如 Menu、Edit Office、Agents、Projects、SMS、Browser。
- 预期结果：工具栏按钮文字随比例变大；按钮仍可点击，文字不互相覆盖。

### CHK-005: 主要弹窗和侧栏随缩放变化

- 关联需求点：主要 DOM 面板受字体缩放影响。
- 验证方法：用 MCP 浏览器工具打开至少一个主要弹窗或侧栏，例如 Agent 面板、设置子区域或其他可打开的主应用弹窗，并在 `120%` 下检查文本。
- 预期结果：弹窗或侧栏中的标题、标签、按钮、说明文本随比例变大；关键操作仍可访问。

### CHK-006: 聊天面板随缩放变化

- 关联需求点：高频阅读区域受字体缩放影响。
- 验证方法：用 MCP 浏览器工具打开聊天面板，在 `100%` 和 `120%` 下检查消息区、输入区、按钮和状态文本。
- 预期结果：聊天面板主要 DOM 文本随比例变大；输入框可输入，按钮可点击，消息内容不被遮挡。

### CHK-007: 项目面板随缩放变化

- 关联需求点：独立样式 `projects.css` 覆盖范围。
- 验证方法：用 MCP 浏览器工具打开 Projects 面板，在 `120%` 下检查项目列表、任务卡片、按钮、详情区域文字。
- 预期结果：项目面板主要 DOM 文本随比例变大；面板仍可滚动，主要按钮和列表项不因字号变化不可用。

### CHK-008: 会议面板随缩放变化

- 关联需求点：会议 DOM 面板覆盖范围。
- 验证方法：用 MCP 浏览器工具打开 Meetings/会议相关面板，在 `120%` 下检查标签页、会议卡片、按钮和表单文字。
- 预期结果：会议面板主要 DOM 文本随比例变大；卡片内容不出现明显重叠，关键按钮可点击。

### CHK-009: 个人偏好持久化

- 关联需求点：当前浏览器/当前用户设置刷新后保留。
- 验证方法：用 MCP 浏览器工具设置为 `120%`，刷新页面，再打开设置面板并读取 DOM 状态或 `localStorage.vo-display-prefs`。
- 预期结果：刷新后控件仍显示 `120%`；页面启动后 DOM UI 仍按 `120%` 显示；偏好存储在本地展示偏好中。

### CHK-010: 不写入全局服务器配置

- 关联需求点：字体大小是个人显示偏好，不影响其他用户。
- 验证方法：设置字体大小后，通过浏览器或接口检查 `/vo-config`，确认服务器配置未新增或修改字体缩放字段。
- 预期结果：字体缩放不写入全局办公室配置；保存其他设置时也不会把字体比例提交为共享配置。

### CHK-011: 默认值和回退行为正确

- 关联需求点：默认值保持现有视觉，异常值不破坏页面。
- 验证方法：用 MCP 浏览器工具清除 `localStorage.vo-display-prefs.fontScale` 后刷新；再手动写入非法值如 `0`、`abc`、`9` 后刷新。
- 预期结果：无本地偏好时回到 `100%`；非法值被归一化到允许范围或回退 `100%`；页面不报错、不白屏。

### CHK-012: 150% 档位布局可用

- 关联需求点：最大档位下布局无明显溢出。
- 验证方法：用 MCP 浏览器工具设置为 `150%`，检查设置面板、工具栏、聊天、项目、会议至少四类区域。
- 预期结果：文本不互相覆盖；关键按钮可点击；需要更多空间的区域可滚动访问。

### CHK-013: 窄屏视口可用

- 关联需求点：兼容性。
- 验证方法：用 MCP 浏览器工具把视口调整到较窄宽度，例如移动或小桌面宽度，在 `150%` 下检查设置面板和至少一个主要面板。
- 预期结果：核心设置和主要操作不被遮挡；文本不出现不可接受的重叠；可通过滚动访问内容。

### CHK-014: 中文界面兼容

- 关联需求点：中英文文案和中文字体兼容。
- 验证方法：用 MCP 浏览器工具切换中文界面，设置 `120%`，检查设置面板、工具栏和一个主要面板。
- 预期结果：显示 `界面字体大小` 等中文文案；中文字体仍使用现有中文字体栈；字号缩放生效；主要文本不乱码、不明显截断。

### CHK-015: Canvas 不被纳入验收

- 关联需求点：非目标范围。
- 验证方法：用 MCP 浏览器工具在 `120%` 下查看办公室画布，同时检查实现说明或设置文案。
- 预期结果：不要求画布内 Agent 名称、状态、家具标签同步变大；设置文案和交付说明明确本期只覆盖 DOM UI。

### CHK-016: MCP 最终验收记录

- 关联需求点：最终验收必须使用 MCP 工具检查。
- 验证方法：开发完成后，在交付说明或测试记录中列出 MCP 浏览器工具检查过的页面状态，包括打开主应用、进入设置、切换档位、刷新持久化、检查关键面板和窄屏视口。
- 预期结果：验收记录能追溯到 MCP 浏览器检查，不只依赖代码阅读或单元测试。
