# App Font Size Scaling Review

## 评审结论

需求方向合理。经过产品澄清，第一版已明确为当前浏览器/当前用户的 DOM UI 字体缩放，不覆盖 canvas 内绘制文字。

当前无阻塞问题，可以进入 checklist。若未来要求覆盖 canvas 绘制文字，应作为独立需求重新评审，因为 canvas 字号分散在 `game.js` 的绘制逻辑中，且与地图缩放、像素风布局、文本测量和点击区域可能有关。

## 产品澄清结果

- 覆盖范围：只覆盖 DOM UI，包括设置、工具栏、主要弹窗、聊天、项目、会议等面板。
- 不覆盖范围：不覆盖办公室画布里的 Agent 名称、状态、家具标签等 canvas 绘制文字。
- 缩放档位：固定档位 `100%`、`110%`、`120%`、`130%`，后续按用户要求追加 `150%`。
- 生效对象：只影响当前浏览器/当前用户，是个人显示偏好，不写入全局办公室配置。
- 验收要求：最终验收必须使用 MCP 浏览器工具检查真实页面。

## 产品层面问题

### P1: “主应用内”范围不明确

主应用包括 DOM UI 和 canvas 办公室画布。用户肉眼看到的 Agent 名称、状态、天气窗口、家具标签等部分文字可能由 canvas 绘制，不受 CSS 字体缩放影响。

建议选项：

- 选项 A：第一版只覆盖 DOM UI，包括设置、工具栏、侧栏、弹窗、聊天、项目、会议、浏览器面板等。
- 选项 B：同时覆盖 canvas 绘制文字，但需要额外改造绘制字体工具函数，并验证画布布局。

建议采用选项 A，先解决主要可读性问题，并在设置文案里命名为 `UI font size` 或 `界面字体大小`。

### P2: 缩放范围需要限制

像素风 UI 中存在 `5px` 到 `18px` 的固定字号，部分面板宽度固定，例如设置面板宽 `270px`。如果允许过大比例，容易导致文本溢出、按钮变形、弹窗不可用。

建议范围：

- `100%` 默认
- `110%`
- `120%`
- `130%`

不建议首版支持任意滑块到 `200%`。

### P3: 是否即时生效

用户期望设置类控件通常能即时预览，但当前 `mmSaveSettings()` 保存后很多设置提示硬刷新。字体缩放如果保存后才生效，体验较差。

建议：

- 控件变化时即时应用预览。
- 点击保存后持久化。
- 如果用户不保存就关闭菜单，可以保持本地预览还是恢复旧值，需要明确。较简单方案是变更即保存到 `localStorage`，保存按钮仍保存其他服务器配置。

## 技术评审

### 架构

当前主应用 CSS 大量使用固定 `px` 字号，`body` 没有全局 `font-size`。可行方案是使用一个全局 CSS 自定义属性：

```css
:root {
  --vo-font-scale: 1;
}
```

并通过 JS 设置：

```js
document.documentElement.style.setProperty('--vo-font-scale', scale);
```

随后将关键 CSS 字号改为：

```css
font-size: calc(11px * var(--vo-font-scale));
```

### 主要风险

- 大量内联样式如 `style="font-size:9px"` 不会被普通 CSS 变量覆盖。
- `projects.css` 有独立体系，应纳入第一版，否则项目面板不会随设置变化。
- JS 动态创建的弹窗和按钮含内联 `style.cssText` 字号，若首版要求全覆盖，需要逐处替换。
- canvas 绘制字体完全不受 CSS 控制，需要单独函数包装，例如 `_scaledCanvasFont('9px Arial')`，工作量更大。
- 字体放大后固定高度、固定宽度区域可能出现溢出，尤其是工具栏、设置面板、左侧 Agent 列表、会议卡片。

### 数据与状态

推荐复用 `vo-display-prefs`：

```json
{
  "fontScale": 1.2
}
```

加载时读取并应用；保存时写回；异常值需归一化到允许范围。

### 权限与安全

无后端权限变化。该设置应仅影响本浏览器本地展示，不写入团队共享服务器配置，避免一个用户改变所有人的字号。

### 兼容性

- 保持默认 `100%`，避免影响现有用户。
- 中文界面使用 `fonts.css` 替换字体族，不应与字号缩放冲突。
- 移动端和窄屏需要验证，因为字号放大会加重溢出。

### 可观测性

不需要服务端可观测性。前端可通过 DOM 检查 `--vo-font-scale` 和 `localStorage.vo-display-prefs.fontScale` 验证。

## 阻塞判断

产品范围已确认，无阻塞问题。本需求定义为 `DOM UI 字体缩放`，不承诺覆盖 canvas 绘制文字和用户自定义内容字号。

如果后续用户要求“画布里的 Agent 名称/状态也必须变大”，应新开或变更需求并重新进入评审。

## 推荐方案

1. 在设置面板增加 `界面字体大小 / UI font size` 控件，使用下拉或分段选项：`100%`、`110%`、`120%`、`130%`、`150%`。
2. 把 `fontScale` 存入 `vo-display-prefs`。
3. 页面启动时尽早应用 `--vo-font-scale`，避免闪烁。
4. 将主要 DOM UI CSS 的固定字号迁移为 `calc(px * var(--vo-font-scale))`。
5. 对常见内联字号优先改为 class 或 CSS 变量；无法一次覆盖的区域记录为后续项。
6. 暂不覆盖 canvas 字体。
7. 最终验收使用 MCP 浏览器工具打开真实页面，检查设置入口、档位切换、刷新持久化、关键 DOM 面板和布局溢出。
