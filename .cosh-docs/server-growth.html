分析与教训

为什么没有拆分模块？

三次大规模膨胀都在同一个 server.py 文件中添加代码，而不是拆分为独立模块。这可能是有意选择 —— 单文件降低了部署复杂度（一个 Python 文件就是全部后端），但会增加新人的学习成本。

当前模块边界

尽管 server.py 很大，但各功能区域的边界相对清晰：配置加载（~200 行）、Agent 发现（~300 行）、HTTP Handler（~6,000+ 行）、WebSocket 代理（~1,500+ 行）、API 端点（~2,000+ 行）。这种"隐式模块化"适合小型团队，但不利于长期维护。
