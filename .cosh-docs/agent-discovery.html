安全边界设计

Hermes 适配器的设计原则是"保守" — 只使用安全的 CLI 表面（profile list, status），不读取 secrets/config/memory 文件。这是一种安全边界设计。

统一花名册

两个 Provider 的结果合并为统一的代理花名册：

状态键（statusKey） 每个代理的唯一标识。OpenClaw 用 agent_id，Hermes 用 hermes:profile
Provider 模式 HermesProvider 类封装 Hermes 的 CLI 交互，OpenClaw 是默认提供者
