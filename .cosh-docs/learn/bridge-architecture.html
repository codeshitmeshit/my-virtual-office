VO Provider Bridge 结构总览

这一页回答“每个 bridge 由哪些文件、类、进程、API 构成”。边界只覆盖 Codex、Claude Code、Hermes 以及公共 runtime/result/SSE 层，不展开 OpenClaw gateway 和项目执行业务。

层次CodexClaudeHermes证据

结构层次

目录职责可以按“路由、adapter、runtime、合同”四个方向读。`app/server.py` 拿到 HTTP 请求并维护后台 run；`app/providers` 内每个 provider 自己处理 profile、workspace、binary、auth/test、chat；`app/provider_app_server.py` 是 Codex 可复用的 JSONL 子进程 runtime；`app/provider_execution.py` 是跨 provider 的结果归一化合同。

路由层

`_handle_codex_run_start`、`_handle_claude_code_run_start`、`_handle_hermes_run_start` 接入 VO API。

适配层

`CodexProvider`、`ClaudeCodeProvider`、`HermesProvider` 对外保持 office agent 形状。

运行层

Codex app-server、Claude CLI、Hermes API/CLI 是真实执行边界。

Codex bridge 构成

Codex 是分层最完整的实现。`app/providers/codex.py` 是 VO adapter，负责发现 agent、读取配置、创建 workspace、chat facade、approval/cancel/compact。`app/providers/codex_app_server.py` 是真正 runtime bridge，内部用 `JsonlAppServerRuntime` 启动 `codex app-server --stdio`。`app/providers/codex_bridge.py` 现在只是 compatibility shim，保留旧 import 路径。

server.py -> CodexProvider -> get_codex_bridge()
-> CodexAppServerClient -> JsonlAppServerRuntime -> codex app-server --stdio
-> CodexHttpBridgeClient -> external /execute, /compact

Claude Code bridge 构成

Claude Code 没有独立 app-server runtime 文件。它的 adapter 和 CLI harness 都在 `app/providers/claude_code.py`：`ClaudeCodeProvider.send_chat_message` 组装 `claude -p --output-format stream-json --verbose --include-partial-messages`，`_ClaudeStreamState` 聚合 stdout 中的 session、reply、tool、usage。`server.py` 的 run wrapper 再把 progress snapshot 转给 ProviderRunBridge。

Hermes bridge 构成

Hermes 是双路径。`HermesProvider` 提供 CLI profile、chat、create/delete/session export 等公共面；同一文件里的 `HermesApiClient` 负责 native API `/health`、`/v1/capabilities`、`/v1/runs`、run SSE、approval、stop。`server.py` 的 `_handle_hermes_chat` 在 API 可用时优先消费 native events，否则回落 CLI chat。

证据清单

结论证据
Codex 真正实现不在 shim`app/providers/codex_bridge.py` re-export；主体在 `app/providers/codex_app_server.py`。
Run/SSE 总线 provider-neutral`app/server.py` `ProviderRunBridge.remember/emit/stream_events` 被 Codex、Claude、Hermes run handler 使用。
结果合同统一`app/provider_execution.py` `normalize_provider_result` 和 `provider_http_status`。
Claude 是 CLI stream-json`app/providers/claude_code.py` 构造 `claude -p --output-format stream-json`。
Hermes API/CLI 双路径`app/providers/hermes.py` 同时包含 `send_chat_message` 和 `HermesApiClient`。
