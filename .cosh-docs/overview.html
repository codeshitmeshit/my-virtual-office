最重要的 Happy Path

用户最核心使用路径：

1. 启动 → Docker Compose 启动 → server.py 三线程初始化 → Gateway Presence 连接
2. 发现 → get_roster() → 双 Provider 发现（OpenClaw + Hermes）→ 构建统一花名册
3. 状态 → Gateway WS 事件流 → Presence 状态机推导 working/idle/finishing
4. 渲染 → 前端 Canvas 渲染办公室 → Agent 在工位间走动
5. 聊天 → 用户打开聊天窗口 → WS 直连 Gateway → 流式响应 + 工具调用展示

3 个必须先理解的核心抽象

1. Gateway Presence 状态机 只读观察者模式，通过 WS 事件推导 Agent 状态（working/idle/finishing），取代旧的 office.py 手动更新。详见 Gateway Presence。
2. 双 Provider 架构 OpenClaw（主路径，文件读取）+ Hermes（适配器，CLI 调用），设计为可扩展 Provider 接口。详见 代理发现。
3. 三线程启动架构 HTTP 主服务器 + WS 代理 + 工作流自动恢复，全部基于 Python 标准库 threading。详见 服务器启动。
